위의 레이아웃의 기본으로 하며, 운영되고 있는 페이지의 마크업을 복사해 해당 수정 부분만 작업한다.

WORK SPACE

마크업, css, image 작업 모두 yt 폴더에서 작업하고, 로컬에서는 http://company.ytlocal.com/로 확인할 수 있으며,
테스트 버전은 http://175.126.125.230/에서 최종 버전 적용은 http://yellotravel.com/에서 확인이 가능하다.

작업 방식
마크업 작업은 개발 작업과 같게 git으로 버전관리를 하며
github.com에서 vagrant-wooripension-dev에 있는 설명과 같게 터미널을 이용하여 vagrant를 설치하여 작업한다.
작업 경로
HTML

페이지가 새로 작업할 경우에 사용되며, 우리펜션 실서비스나 운영 페이지의 레이아웃 구성을 가져와서 작업한다.
yt  >  vagrant-wooripension-dev  >  www



CSS

css는 크게 bootstrap.css, font-awesome.css, layout.css로 적용되어 있으며,
실제 작업은 layout.css에서만 하며 bootstrap.css에서 수정해야될 부분은 클래스를 가져와 오버라이딩해서 작업하는 것을 원칙으로 한다.
yt  >  vagrant-wooripension-dev  >  www  >  css


기존 콘텐츠와 관련된 콘텐츠를 작업할 경우에는 해당 콘텐츠의 css 위치를 찾아 하단에 날짜 주석을 기입해 처리하고,

/* gnb */
#gnb {overflow:hidden;..}
#gnb li {float:left;..}

/* 20150801 추가 시작 */
#gnb .link_gnb {display:block;..}
#gnb .link_gnb .ico_gnb {width:6px;..}
/* 20150801 추가 끝 */

      .. OR ..

#gnb .link_gnb {display:block;..} /* 20150801 추가 */
#gnb .link_gnb .ico_gnb {width:6px;..} /* 20150801 추가 */


새로운 콘텐츠 작업의 경우에는 layout.css의 하단에 해당 부분의 설명과 날짜를 주석 처리하여 추가한다.
/* 200150801 side 영역 닫기/열기 */
.cont_admin .admin_content {margin-left:230px;} /* default - side 열기 */
.sidemenu_close .cont_admin .admin_content {margin-left:0;} /* sidemenu_close 클래스 추가시 side 닫기 */
